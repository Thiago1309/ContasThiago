git clone ** -> copiar o codigo da branch master do repositório passado

git branch -> lista as branches criadas marcando a que está sendo usada

git checkout -b <nome> -> cria e seleciona um branch

git checkout <nome> -> seleciona a branche passada em <nome>

git status -> exibe os arquivos que sofreram alterações e não foram comitados

git add --all -> adiciona à fila de comit todos os arquivos que sofreram alterações

git add <arquivo> -> adiciona à fila de comit os arquivos que listados como parâmetro

git commit -am <mensagem> -> criação de um bloque de arquivos À serem submetidos ([-am] a = adiciona os arquivos ainda não inserios / m = somente os arquivos modificados) <mensagem> referencia a mensagem que será exibida no repositorio

git push -> submete ao repositorio os arquivos empacotados

git push --set-upstream origin dev -> cria a branch no repositorio (utilizar caso receba erro de não existência desta)

git pull -> baixa os arquivos do repositorio
